## 3.1 Анализ полноты каталога

Перед применением статистических методов необходимо определить **магнитуду полноты** Mc — минимальную магнитуду, выше которой каталог считается репрезентативным.

### 3.1.1 Метод максимальной кривизны (Wiemer & Wyss, 2000)

Метод основан на нахождении точки максимальной кривизны (второй производной) кумулятивной частотно-магнитудной зависимости (FMD).

**Алгоритм:**
1. Построить некумулятивную гистограмму числа событий N(M) с шагом ΔM = 0.1.
2. Найти магнитуду максимума некумулятивного распределения: Mc = argmax N(M).
3. Эмпирическая поправка: Mc_corrected = Mc + 0.2 (систематическое смещение метода).

### 3.1.2 MLE-оценка b-value (Shi & Bolt, 1982)

Закон Гутенберга–Рихтера: log10(N) = a − b·M

**Формула MLE (максимального правдоподобия):**

    b = log10(e) / (M_mean - (Mc - ΔM/2))

где:
- M_mean = среднее значение магнитуд в выборке (M >= Mc)
- Mc = магнитуда полноты
- ΔM = шаг бинирования (0.1)
- log10(e) ≈ 0.4343

**Стандартная ошибка b-value (Shi & Bolt, 1982):**

    σ_b = 2.30 · b^2 · sqrt(sum((Mi - M_mean)^2) / (N · (N-1)))

### 3.1.3 Матрица полноты по регионам и эпохам

Для учёта пространственно-временной неоднородности каталога строится матрица Mc[region, epoch]:

| Регион (FE) | 1900–1939 | 1940–1963 | 1964–1975 | 1976–2004 | 2005–2024 |
|-------------|-----------|-----------|-----------|-----------|-----------|
| Circum-Pacific | 7.0 | 6.5 | 6.0 | 5.5 | 5.0 |
| Alpide Belt | 7.0 | 6.5 | 6.0 | 5.5 | 5.0 |
| Mid-Ocean Ridges | 7.5 | 7.0 | 6.5 | 6.0 | 5.5 |
| Intraplate | 7.5 | 7.0 | 6.5 | 6.0 | 5.5 |

Матрица используется для фильтрации каталога: события включаются в анализ только при M >= Mc[region, epoch].

## 3.2 Тектоническое расстояние

### Обоснование выбора

Евклидово (или сферическое геодезическое) расстояние не отражает реальную физическую связанность сейсмических событий. Два события, разделённых океаном, но расположенных на одном разломе (например, на Тихоокеанском кольце), физически ближе, чем два события, разделённых континентом.

**Тектоническое расстояние** r_tect(i, j) — длина кратчайшего пути между событиями i и j по графу активных разломов. Оно учитывает структуру литосферы и корреляцию вдоль активных тектонических структур.

### Граф NetworkX: структура

**Построение графа:**
- **Узлы**: точки дискретизации разломов и границ плит (шаг ~10 км), а также пересечения разломов.
- **Рёбра**: сегменты разломов между соседними узлами.
- **Вес ребра**: геодезическое расстояние в км (`geopy.distance.geodesic`).
- **Атрибуты узла**: `lat`, `lon`, `fault_id`, `plate_boundary_type` (subduction/transform/rift).
- **Итого**: ~180 000 узлов, ~190 000 рёбер.

```python
import networkx as nx
from geopy.distance import geodesic

G = nx.Graph()

for fault in gem_faults:
    coords = fault["geometry"]["coordinates"]
    for k in range(len(coords) - 1):
        node_a = tuple(coords[k])      # (lon, lat)
        node_b = tuple(coords[k + 1])
        G.add_node(node_a, lat=node_a[1], lon=node_a[0])
        G.add_node(node_b, lat=node_b[1], lon=node_b[0])
        dist_km = geodesic((node_a[1], node_a[0]),
                           (node_b[1], node_b[0])).km
        G.add_edge(node_a, node_b, weight=dist_km)
```

### Алгоритм Дейкстры

Для каждой пары событий (i, j) кратчайший путь вычисляется алгоритмом Дейкстры:

```python
r_tect = nx.shortest_path_length(G, source=node_i, target=node_j, weight='weight')
```

Результаты кэшируются: матрица расстояний хранится в разреженном формате (`scipy.sparse`).

### Фолбэк для внутриплитных событий

Если ближайший узел графа находится дальше **500 км** от события (внутриплитная сейсмичность), используется фолбэк:

    r_tect_fallback(i, j) = r_geodesic(i, j) * 1.5

Коэффициент 1.5 эмпирически откалиброван на данных континентальной сейсмичности (Индостанская плита, Северо-Американская платформа).

## 3.3 Метрика ближайшего соседа

### Вывод формулы

Метрика η ("эта") введена Baiesi & Paczuski (2004) для измерения "близости" двух сейсмических событий в пространственно-временном масштабированном пространстве.

**Полная формула:**

    η_ij = t_ij · r_ij^df · 10^(−b · m_i)

где:
- **η_ij** — метрика близости события j к событию i (безразмерная)
- **t_ij = t_j − t_i** — временной интервал между событиями i и j (в годах); t_j > t_i (причинность)
- **r_ij** — тектоническое расстояние между i и j (в км)
- **df** — фрактальная размерность распределения эпицентров (типично df ≈ 1.6 для глобального каталога)
- **b** — наклон закона Гутенберга–Рихтера (b ≈ 1.0)
- **m_i** — магнитуда события-предшественника i

**Физический смысл каждого члена:**
- `t_ij`: чем дольше временной интервал, тем менее вероятна причинная связь.
- `r_ij^df`: учёт фрактальной геометрии сейсмичности; пространственное "отдаление" масштабируется нелинейно.
- `10^(−b·m_i)`: нормировка на магнитуду источника; более сильное землетрясение имеет большую "зону влияния", поэтому η уменьшается (нормировка вниз).

**Ближайший родитель** события j — событие i*, минимизирующее η_ij среди всех предшественников:

    i* = argmin_{i: t_i < t_j} η_ij

### Обоснование параметров

- **df = 1.6**: оценка фрактальной размерности распределения эпицентров методом корреляционной интегральной функции (Grassberger–Procaccia) на глобальном каталоге ISC 1964–2023, M≥5.
- **b = 1.0**: стандартное значение для глобального каталога. Уточняется по регионам с помощью `CompletenessAnalyzer`.

### Отличие от оригинала Baiesi & Paczuski (2004)

| Аспект | Baiesi & Paczuski (2004) | Данный проект |
|--------|--------------------------|---------------|
| Расстояние r | Евклидово (2D) | Тектоническое (граф разломов) |
| Каталог | Южная Калифорния | Глобальный (USGS + ISC + NOAA) |
| df | 1.6 (региональный) | 1.6 (глобальный, переоценён) |
| Единицы r | км | км |
| Обработка исторических данных | — | Взвешивание по quality_score |

## 3.4 Идентификация глобальных серий

### Алгоритм скользящего окна

Скользящее окно шириной W (в годах) смещается по временной оси с шагом S:

- **Ширина окна W**: 1 год (365 дней)
- **Шаг S**: 0.25 года (90 дней)

### Критерии серии

Окно считается **глобальной серией**, если в нём выполнены все три условия:

1. **N ≥ 4** событий с M ≥ 6.5 и η < η_threshold (η_threshold определяется из распределения η).
2. **Максимальная магнитуда M_max ≥ 6.5** в пределах окна.
3. **≥ 3 различных регионов Флинна–Энгдала** (FE regions) задействованы — критерий "глобальности".

### Псевдокод алгоритма

```python
import pandas as pd
import numpy as np
from typing import List

def find_global_series(
    catalog: pd.DataFrame,
    eta_matrix: np.ndarray,
    eta_threshold: float,
    window_years: float = 1.0,
    step_years: float = 0.25,
    min_events: int = 4,
    min_magnitude: float = 6.5,
    min_fe_regions: int = 3,
) -> List[dict]:
    """
    Поиск глобальных сейсмических серий скользящим окном.

    Параметры
    ---------
    catalog : pd.DataFrame
        Унифицированный каталог (содержит: year_decimal, magnitude,
        flinn_engdahl_region, event_id).
    eta_matrix : np.ndarray
        Матрица метрик η[i, j] (верхнетреугольная).
    eta_threshold : float
        Порог для определения связанных событий.
    window_years : float
        Ширина скользящего окна в годах.
    step_years : float
        Шаг скользящего окна в годах.
    min_events : int
        Минимальное число связанных событий в серии.
    min_magnitude : float
        Минимальная магнитуда для включения в серию.
    min_fe_regions : int
        Минимальное число различных регионов Флинна-Энгдала.

    Возвращает
    ----------
    List[dict]
        Список обнаруженных серий с метаданными.
    """
    series_list = []
    df = catalog[catalog["magnitude"] >= min_magnitude].copy()
    df["year_decimal"] = df["year"] + (df["month"] - 1) / 12

    t_start = df["year_decimal"].min()
    t_end = df["year_decimal"].max() - window_years

    t = t_start
    while t <= t_end:
        window_mask = (
            (df["year_decimal"] >= t) &
            (df["year_decimal"] < t + window_years)
        )
        window_events = df[window_mask]

        if len(window_events) < min_events:
            t += step_years
            continue

        # Фильтрация по η: оставляем события с η < threshold к любому соседу
        indices = window_events.index.tolist()
        connected = []
        for idx in indices:
            eta_vals = eta_matrix[idx, indices]
            if np.any(eta_vals < eta_threshold):
                connected.append(idx)

        if len(connected) < min_events:
            t += step_years
            continue

        fe_regions = df.loc[connected, "flinn_engdahl_region"].nunique()
        if fe_regions < min_fe_regions:
            t += step_years
            continue

        series_list.append({
            "window_start": t,
            "window_end": t + window_years,
            "event_ids": df.loc[connected, "event_id"].tolist(),
            "n_events": len(connected),
            "max_magnitude": df.loc[connected, "magnitude"].max(),
            "n_fe_regions": fe_regions,
        })
        t += step_years

    # Удаление перекрывающихся серий (сохранить наибольшую)
    series_list = merge_overlapping_series(series_list)
    return series_list
```

## 3.5 Монте-Карло тестирование

### Нулевая гипотеза

**H0**: Глобальные сейсмические серии являются результатом случайного распределения событий во времени (временная однородность при сохранении пространственного распределения).

Иными словами: наблюдаемая кластеризация в η-пространстве не превышает ожидаемую при случайном временном порядке событий.

### Процедура перестановки

1. Зафиксировать набор событий (координаты, магнитуды) из реального каталога.
2. **Случайно перемешать временны́е метки** (годы событий) — времена переприсваиваются случайным образом среди тех же событий.
3. Пересчитать η-матрицу для перемешанного каталога.
4. Применить алгоритм поиска серий и зафиксировать число обнаруженных серий N_sim.
5. Повторить шаги 2–4 **1000 раз** → получить нулевое распределение N_null.

### Интерпретация p-value

    p-value = P(N_null >= N_obs)
            = (число симуляций с N_sim >= N_obs) / 1000

| p-value | Интерпретация |
|---------|--------------|
| p < 0.01 | Высокозначимый результат; H0 отвергается |
| 0.01 ≤ p < 0.05 | Значимый результат |
| 0.05 ≤ p < 0.10 | Слабозначимый; требует осторожной интерпретации |
| p ≥ 0.10 | Нет оснований для отклонения H0 |

**Важно**: малое p-value свидетельствует о статистической значимости **временно́й кластеризации**, но не устанавливает физическую причинно-следственную связь между событиями в разных регионах. Интерпретация результатов обсуждается в разделе [06_results_interpretation.md](06_results_interpretation.md).
